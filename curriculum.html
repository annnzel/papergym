TEAM CURRICULUM

16주 후, AI 변화에 휘둘리지 않는 팀

PaperGym v2는 연구자 양성 과정이 아닙니다. 비개발자 AI 빌더, 마케터, 기획자, 오퍼레이터가 13편의 핵심 논문으로 기본기를 만들고, 매주 PaperGym이 지정한 보강 자료를 그 위에 연결하는 팀 스터디입니다. 새로운 모델·도구·뉴스가 나와도 “이게 뭔지, 왜 중요한지, 우리 일에 쓸 만한지”를 판단할 수 있게 만듭니다.

SELECTION CRITERIA

📐 4가지 선정 기준

📜
1. 1차 소스 (Primary Source)

NeurIPS · ICLR · OpenAI · Anthropic · DeepMind 공식 publication만. 블로그 요약·해설 X. 모든 후속 논문의 *baseline*이 되는 원조.

⏳
2. Classic (시간 견딘 정전)

최소 1년 이상 인용 누적 + 후속 연구의 *기준점*으로 작동. *한때 인기*가 아니라 *교과서급 정전*. 5,000+ citation 기본.

🔧
3. 지금 도구의 뿌리

Claude Code·ChatGPT·MCP·RAG의 *origin 논문*. 새 모델 발표를 이 13편 중 어디 계열인지 *분류 가능*해야 함.

🏗️
4. 비개발자 AI 빌더 관점

연구자 양성 X. 수식 완전 정복보다 *업무 판단력*이 목표. 각 논문은 제안서, 콘텐츠, 리서치, 자동화, 에이전트 설계에 연결된다.

WEEK 0

🧭 시작 전 1시간: LLM 최소 기초

CoT, ReAct, RLHF를 바로 읽기 전에 팀 전체가 같은 언어를 가져야 합니다. Week 0은 깊은 수학이 아니라 논문을 읽을 때 길을 잃지 않기 위한 최소 개념 부트캠프입니다.

알아야 할 10개 단어
  • Token — 모델이 읽는 텍스트 조각
  • Embedding — 토큰을 숫자 공간에 놓는 방식
  • Attention — 토큰 사이 관계를 계산하는 구조
  • Transformer — 현대 LLM의 기본 뼈대
  • Pretraining — 다음 토큰 예측을 배우는 단계
  • Post-training — assistant처럼 행동하게 다듬는 단계
  • RLHF — 사람 선호를 반영하는 훈련
  • Context window — 모델이 한 번에 보는 작업 공간
  • Inference — 학습된 모델이 답을 생성하는 과정
  • Benchmark — 성능 시험. 실무 성능과 다를 수 있음
통과 질문
  1. LLM은 검색엔진인가, 예측기인가?
  2. Pretraining과 post-training은 무엇이 다른가?
  3. Context window와 memory는 무엇이 다른가?
  4. AI agent는 왜 도구를 호출하는가?
  5. Benchmark 점수가 높으면 실무에서도 무조건 좋은가?

THE 13 PAPERS

📚 13편 풀 가이드

Phase 1 · Form Building (Week 1-4)

W1

Chain-of-Thought Prompting

Wei et al., NeurIPS 2022 · 15,000+ citation

핵심: 답에 이르는 *중간 추론 단계*를 예시로 보여주면 큰 모델의 추론 능력이 극적으로 드러난다.

왜 정전: 모든 reasoning 발전형(Self-Consistency·ToT·o1·R1)의 *baseline*. "Let's think step by step" 한 줄이 산업 전반 채택.

최신 보강: Self-Consistency, o1/R1, test-time compute를 함께 본다.

팀 산출물: 복잡한 업무를 단계별 사고 구조로 쪼개는 CoT 프롬프트 카드.

다음과 연결: CoT는 *생각만*. 다음 ReAct는 *생각 + 행동* 추가.

W2

ReAct: Synergizing Reasoning and Acting

Yao et al., ICLR 2023 · 4,000+ citation

핵심: Thought → Action → Observation 루프. 모델이 *생각하고 행동하고 관찰*하며 외부 정보를 활용.

왜 정전: Claude Code · Cursor · Devin 등 *모든 coding agent의 작동 원리*. Agent 패러다임의 시작점.

최신 보강: Claude Code, Cursor, Devin 같은 coding agent 사례와 연결한다.

팀 산출물: Thought → Action → Observation 업무 루프 카드.

다음과 연결: ReAct는 *도구를 어떻게 쓰는지* 보여줌. 다음 Toolformer는 *언제 어떤 도구를 써야 하는지* 학습.

W3

Toolformer: LMs Can Teach Themselves to Use Tools

Schick et al., Meta AI 2023 · 2,500+ citation

핵심: 모델이 자기지도로 *언제 어떤 API를 호출할지* 학습. 계산기·검색·번역 등 6개 도구.

왜 정전: Function calling · MCP · LangChain tool use의 *학문적 뿌리*. 모델 + 도구 패러다임의 정전.

최신 보강: Function calling, MCP, computer use, tool router 개념과 연결한다.

팀 산출물: 도구 사용 기준표: 언제 검색·계산·파일·문서 도구가 필요한가.

다음과 연결: Phase 1은 *agent 작동*. 다음 Phase 2는 *왜 agent가 친절한 assistant인가*.

W4

Phase 1 Review · Agent Loop Map

CoT · ReAct · Toolformer 통합

활동: 세 논문을 비교해 AI agent가 왜 생각하고, 행동하고, 관찰하는지 한 장으로 정리한다.

팀 산출물: Workflow로 충분한 일과 agent가 필요한 일을 구분하는 체크리스트.

성공 기준: Claude Code 같은 도구를 단순 챗봇이 아니라 agentic workflow로 설명할 수 있다.

Phase 2 · Assistant Training (Week 5-8)

W5

InstructGPT / RLHF

Ouyang et al., OpenAI 2022 · 12,000+ citation

핵심: SFT + Reward Model + PPO 3단계로 base model을 instruction-following assistant로 변환.

왜 정전: ChatGPT의 *직접적 부모*. 모든 modern assistant 모델 (Claude·Gemini·LLaMA Chat)의 기본 레시피.

최신 보강: 최신 모델 release note를 post-training 관점에서 읽는다.

팀 산출물: Base model과 assistant model 차이를 설명하는 1페이지 카드.

다음과 연결: InstructGPT는 *human feedback*. 다음 CAI는 *AI feedback + 명시적 원칙*.

W6

Constitutional AI

Bai et al., Anthropic 2022 · 3,000+ citation

핵심: 명시적 *원칙 목록(constitution)*을 기준으로 AI가 자기 답변을 비판·수정. RLAIF (AI feedback).

왜 정전: *Claude의 디자인 철학 그 자체*. "원칙 기반 alignment"의 정전. 기업용 AI policy 설계의 baseline.

최신 보강: 기업 AI policy, brand safety, refusal behavior와 연결한다.

팀 산출물: 팀용 AI 사용 원칙 constitution 초안.

다음과 연결: Phase 2 마지막은 *실패에서 배우는 assistant* — Reflexion.

W7

Reflexion: Verbal Reinforcement Learning

Shinn et al., NeurIPS 2023 · 1,500+ citation

핵심: 실패를 *자연어로 reflection*해서 memory에 저장 → 다음 시도 개선. 모델 재훈련 없이 *행동 개선*.

왜 정전 진입: Agent self-improvement의 *대표 사례*. Claude Code의 verification 루프 설계 origin.

최신 보강: Agent verification, self-correction, eval loop와 연결한다.

팀 산출물: 실패 후 개선 메모 템플릿: 무엇을 시도했고 무엇을 배웠나.

다음과 연결: Phase 1-2는 *agent + assistant 사용*. 다음 Phase 3은 *그 모델이 어떻게 만들어졌나*.

W8

Phase 2 Review · Assistant Training Map

InstructGPT · CAI · Reflexion 통합

활동: base model이 assistant가 되는 과정을 instruction tuning, feedback, principle, reflection으로 분해한다.

팀 산출물: 좋은 AI 답변을 평가하는 기준표: 유용성, 정확성, 안전성, 톤, 재시도 가능성.

성공 기준: AI의 친절함과 조심스러움이 성격이 아니라 post-training의 결과임을 설명할 수 있다.

Phase 3 · LLM Foundation (Week 9-12)

W9

Attention Is All You Need

Vaswani et al., NeurIPS 2017 · 120,000+ citation

핵심: Recurrence/Convolution 없이 *attention*만으로 sequence 처리. Transformer 구조 제안.

왜 정전: 현대 LLM의 *뼈대*. AI 역사상 가장 인용 많은 논문 중 하나. *모든* AI 교과서 첫 챕터.

최신 보강: Transformer visual guide와 최신 architecture 요약을 함께 본다.

팀 산출물: Attention을 비전문가에게 설명하는 3문장 비유.

다음과 연결: Transformer 구조 → 다음 GPT-3는 *그 구조를 어디까지 키울 수 있나*.

W10

GPT-3 / Language Models are Few-Shot Learners

Brown et al., NeurIPS 2020 · 35,000+ citation

핵심: 175B 파라미터 + few-shot prompting. fine-tuning 없이 다양한 과제 수행 가능.

왜 정전: "범용 LLM" 패러다임의 *시작*. 프롬프트 엔지니어링 분야 탄생의 직접 원인.

최신 보강: In-context learning과 prompt example design을 함께 본다.

팀 산출물: 좋은 예시 3개로 결과 품질을 끌어올리는 few-shot 템플릿.

다음과 연결: GPT-3는 *크면 좋다*. 다음 Scaling Laws는 *왜 그런가*.

W11

Scaling Laws for Neural Language Models

Kaplan et al., OpenAI 2020 · 8,000+ citation

핵심: 모델 크기·데이터·compute의 power-law 관계. 큰 모델이 *sample-efficient*.

왜 정전: *AI 투자 곡선*을 정의. 모든 frontier lab의 *roadmap 기반*.

최신 보강: AI capex, compute race, frontier lab 발표를 함께 읽는다.

팀 산출물: 모델 발표 읽기 체크리스트: scale, data, compute, benchmark, cost.

다음과 연결: Scaling Laws는 *큰 모델이 좋다*. 다음 Chinchilla는 *그 직관의 보정*.

W12

Chinchilla / Compute-Optimal Training

Hoffmann et al., DeepMind 2022 · 5,000+ citation

핵심: 모델 크기를 두 배로 늘릴 때 *훈련 데이터*도 두 배. 기존 모델은 *undertrained*.

왜 정전: Scaling Laws의 *재해석*. Llama·Mistral 등 *모든 efficient* open model의 baseline.

최신 보강: LLaMA, Mistral, Qwen, DeepSeek 등 open model 흐름과 연결한다.

팀 산출물: 용도별 모델 선택 기준표: 성능, 비용, 보안, 속도, 맥락 길이.

다음과 연결: Phase 3은 *어떻게 만드나*. 다음 Phase 4는 *어떻게 *기억*하나*.

Phase 4 · Memory & Synthesis (Week 13-16)

W13

RAG / Retrieval-Augmented Generation

Lewis et al., NeurIPS 2020 · 8,000+ citation

핵심: Parametric memory(모델 가중치) + Non-parametric memory(외부 검색) 결합.

왜 정전: 모든 *enterprise AI assistant* (Notion AI · Copilot · Perplexity)의 baseline. RAG = AI 도입 진입점.

최신 보강: Enterprise RAG, Perplexity, Notion AI, Copilot류 사례와 연결한다.

팀 산출물: RAG가 필요한 업무와 단순 prompting으로 충분한 업무를 구분하는 표.

다음과 연결: RAG는 *검색*. 다음 MemGPT는 *기억 관리*.

W14

MemGPT: LLMs as Operating Systems

Packer et al., UC Berkeley 2023 · 1,200+ citation

핵심: LLM을 OS처럼 — context window = main memory, 외부 저장소 = disk. *기억 관리* 패러다임.

왜 정전 진입: *Memory engineering* 분야의 origin. Claude Memory · Letta · Zep 등 후속의 baseline.

최신 보강: Letta, Zep, Graphiti, Claude Memory, context engineering과 연결한다.

팀 산출물: 팀 지식 저장소 구조도: session, docs, memory, archive의 차이.

다음과 연결: MemGPT는 *기억*. 다음 ToT는 *생각의 탐색* — 마지막 reasoning 발전.

W15

Tree of Thoughts

Yao et al., NeurIPS 2023 · 2,000+ citation

핵심: 단일 chain-of-thought → 여러 thought 후보를 *tree 탐색*. 평가 + backtrack.

왜 정전 진입: Test-time compute (o1·R1) 시대의 *baseline*. Reasoning 발전형의 정전.

최신 보강: o1/R1, self-consistency, test-time compute와 연결한다.

팀 산출물: 중요한 의사결정을 여러 경로로 탐색하는 decision prompt template.

다음과 연결: 마지막 W16은 *Final Synthesis* — 13편 전체를 하나의 지도로 묶음.

W16

🏆 Final Synthesis · Graduation

전체 통합 주차

활동: 13편을 *하나의 지도*로 통합. AI Paper Lineage Map 작성. PaperGym Graduation Note.

최신 보강: 최근 1개월 AI 뉴스 5개를 직접 분류한다.

팀 산출물: AI Paper Lineage Map + 팀별 업무 적용 지도 + graduation note.

성공 기준: 새 AI 모델 발표를 보면 *Phase 1-4 중 어디 계열인지 즉시 분류 가능*.

이때부터 PaperGym은 *수동적 학습 도구*에서 *능동적 분류 도구*로 바뀝니다.
