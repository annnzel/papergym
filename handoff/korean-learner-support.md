# PaperGym — 한국어 학습자 지원 설계

> 목적: 공식 한국어 번역이 거의 없는 AI 논문을 한국어 학습자가 원문 중심으로 읽을 수 있게 돕는다.  
> 핵심 원칙: **원문을 대체하지 않고, 원문으로 들어가는 발판을 제공한다.**

---

## 1. 공식 한국어판은 있나?

대부분의 핵심 AI 논문은 **공식 한국어판이 없다**고 보는 것이 안전하다.

| 자료 유형 | 한국어 공식판 가능성 | 설명 |
|---|---:|---|
| arXiv 논문 | 거의 없음 | 원문은 영어 PDF가 기준 |
| NeurIPS / ICLR / ICML 논문 | 거의 없음 | 학회 공식 언어는 영어 |
| OpenAI / Anthropic 연구 논문 | 거의 없음 | 공식 블로그 일부만 다국어 가능성이 있으나 논문은 영어 |
| 기업 블로그 | 일부 가능 | 제품/뉴스 페이지는 번역될 수 있지만 연구 원문은 영어 우선 |
| 한국어 해설 콘텐츠 | 있음 | 공식 번역이 아니라 2차 해설로 봐야 함 |

따라서 PaperGym의 한국어 전략은 “한국어 번역본 제공”이 아니라:

> **영어 원문을 한국어 사고로 소화하게 만드는 bilingual scaffold**

가 되어야 한다.

---

## 2. 한국어 학습자에게 진짜 필요한 것

한국어 학습자가 논문에서 막히는 지점은 영어 자체보다 아래 5개다.

1. **논문 구조를 모름**  
   Abstract, Introduction, Method, Experiment 중 어디를 봐야 할지 모른다.

2. **용어가 한 번에 안 들어옴**  
   alignment, reasoning, retrieval, reward model 같은 단어가 계속 걸린다.

3. **수식/실험표에서 멈춤**  
   30분 학습에서 건너뛰어도 되는 부분과 봐야 하는 부분을 구분하지 못한다.

4. **읽었는데 내 말로 못 바꿈**  
   영어 문장을 이해한 것 같지만 한국어로 설명하려면 막힌다.

5. **업무 연결이 안 됨**  
   “그래서 Claude Code / PR / 제안서랑 무슨 상관?”까지 못 간다.

PaperGym은 이 5개를 해결해야 한다.

## 3. 한국어 학습자용 4단계 보조 레이어

### Layer 1. Korean Warm-up

원문 읽기 전에 한국어로 3가지만 알려준다.

```text
오늘 이 논문에서 볼 것:
1. 이 논문이 해결하려는 문제
2. 이 논문이 제안한 새 구조
3. Claude/ChatGPT/Agent와 연결되는 지점
```

주의: 여기서 전체 요약을 다 주면 안 된다.  
PaperGym의 원칙은 “떠먹이기”가 아니라 “읽을 준비운동”이다.

---

### Layer 2. Bilingual Keyword List

논문별 핵심 용어 8~12개를 영어/한국어/쉬운 설명으로 제공한다.

예:

| English | 한국어 | 쉬운 설명 |
|---|---|---|
| reasoning | 추론 | 답을 바로 내는 게 아니라 중간 판단 과정을 거치는 것 |
| action | 행동 / 도구 호출 | 검색, 계산, 파일 읽기처럼 모델 바깥의 일을 실행하는 것 |
| observation | 관찰 결과 | 도구를 쓴 뒤 돌아온 결과. 다음 판단의 재료 |

---

### Layer 3. Read-This-Not-That

30분 안에 볼 것과 건너뛸 것을 명확히 나눈다.

```text
오늘 볼 것:
- Abstract
- Figure 1
- Introduction 첫 2페이지
- Method의 큰 흐름

오늘 건너뛸 것:
- 상세 수식
- 전체 실험표
- Appendix
- 모든 관련연구 문단
```

---

### Layer 4. Korean Recall Test

읽고 돌아온 뒤에는 한국어로 답하게 한다.

예:

```text
영어 원문을 덮고, 한국어로 답하세요.
1. 이 논문이 해결하려는 문제는 무엇인가요?
2. 이 논문이 제안한 방법은 기존 방식과 무엇이 다른가요?
3. 내 업무에 적용한다면 어디에 쓸 수 있나요?
```

중요: 답변 평가는 한국어로 하되, 핵심 영어 용어는 살린다.

## 4. PaperGym 앱에 넣을 한국어 모드

### 모드 이름

```text
Korean Scaffold Mode
```

한국어:

```text
한국어 보조 모드
```

### 설명 문구

```text
원문은 영어로 읽고, 이해의 발판은 한국어로 제공합니다.
요약을 대신 읽어주는 모드가 아니라, 원문을 직접 들 수 있게 자세를 잡아주는 모드입니다.
```

### UI 토글

```text
[ ] Korean Scaffold
```

또는

```text
한국어 보조 켜기
```

### 켜졌을 때 추가되는 것

1. 한국어 3문장 워밍업
2. 영어-한국어 핵심 용어표
3. 오늘 볼 것 / 건너뛸 것
4. 한국어 회상 질문
5. Ann 업무 연결 문장

## 5. 공식 번역이 없다는 점을 제품 강점으로 바꾸기

약점처럼 보일 수 있지만, 오히려 PaperGym의 포지셔닝에 맞다.

나쁜 표현:

> 공식 한국어 번역이 없어서 AI가 번역해드립니다.

좋은 표현:

> 공식 한국어판이 거의 없기 때문에, PaperGym은 원문을 버리지 않고 한국어 발판을 붙입니다.

더 좋은 표현:

> 논문은 영어 원문으로 들고, 이해는 내 언어로 고정합니다.

제품 카피:

```text
Read the paper in English.
Lock the concept in Korean.
```

한국어:

```text
논문은 영어로 읽고,
개념은 한국어로 박아 넣습니다.
```

## 6. 한국어 학습자용 S1 Prescription 추가 필드

기존 S1 JSON에 아래 필드를 추가하면 된다.

```json
{
  "korean_scaffold": {
    "warmup_ko": [
      "이 논문은 무엇을 다루는지 1문장",
      "왜 지금 읽는지 1문장",
      "오늘 어디까지만 이해하면 되는지 1문장"
    ],
    "key_terms": [
      {
        "en": "reasoning",
        "ko": "추론",
        "plain_ko": "답을 바로 내지 않고 중간 판단 과정을 거치는 것"
      }
    ],
    "read_this": ["Abstract", "Figure 1", "§1 Introduction"],
    "skip_this": ["Appendix", "detailed equations", "full experiment tables"],
    "recall_questions_ko": [
      "이 논문이 해결하려는 문제는 무엇인가요?",
      "핵심 아이디어를 한국어 한 문장으로 말하면?",
      "내 업무와 연결되는 지점은?"
    ]
  }
}
```

## 7. 한국어 학습자를 위한 평가 기준

PaperGym은 영어 독해 점수를 매기지 않는다.  
한국어 학습자에게 중요한 것은 아래다.

| 평가 항목 | 질문 |
|---|---|
| 원문 접촉 | 실제 영어 원문을 봤는가? |
| 구조 이해 | 문제-방법-결과를 구분했는가? |
| 용어 고정 | 핵심 영어 용어를 한국어로 설명할 수 있는가? |
| 자기 언어화 | 논문을 덮고 내 말로 설명할 수 있는가? |
| 업무 연결 | Claude/업무/제안서/콘텐츠와 연결했는가? |

앱 카피:

```text
We don't grade your English.
We grade whether the idea survived translation into your own language.
```

한국어:

```text
영어 실력을 채점하지 않습니다.
개념이 내 언어로 살아남았는지를 봅니다.
```

---

## 8. 구현 우선순위

해커톤 MVP에서는 아래 3개만 넣어도 충분하다.

1. **Korean Warm-up 3문장**
2. **핵심 용어표 5개**
3. **한국어 회상 질문 3개**

나머지는 이후 확장.

---

## 9. 한 줄 결론

> 공식 한국어 논문을 찾는 대신, PaperGym은 영어 원문 위에 한국어 scaffold를 얹는다.  
> 목표는 번역본 소비가 아니라, 원문 개념을 내 언어로 고정하는 것이다.
